xv6的系统调用，详见bookriscv-rev1-chinese.pdf的P7

1.1 Processes and memory

xv6上的系统调用如下：

在xv6上使用系统调用，一般要用到以下头文件：

#include "kernel/types.h"
#include "user/user.h"
#include "kernel/fcntl.h"

fork系统调用可以为父进程创建一个子进程，创建成功时，为子进程返回0，为父进程返回子进程的PID，失败时返回-1。

int fork();

exit系统调用退出调用进程，并释放资源，如内存和打开的文件。exit需要一个整形参数，一般来说，0表示成功，1表示失败。

int exit(int status);

wait系统调用返回一个当前进程的已退出的子进程的PID，并将其退出状态复制到指针addr所指的地址上；若调用进程的子进程都没有退出，则等待一个子进程退出。若调用者没有子进程，则返回-1；若不关心子进程的退出状态，可以传一个0地址作为参数。

int wait(uint64 addr);

实现一个xv6的用户应用test，编写/user/test.c，在Makefile的UPROGS中加入$U/_test\，验证以上系统调用的功能：

#include "kernel/types.h"
#include "user/user.h"
#include "kernel/fcntl.h"

int main(int argc, char* argv[])
{
    int pid;
    pid = fork();

    if (pid > 0)
    {
        wait(0);
        printf("Here is parent process.\n");
        printf("child %d is done.\n", pid);
    }
    else if (pid == 0)
    {
        printf("Here is child process.\n");
        exit(0);
    }
    else if (pid == -1)
    {
        printf("fork() error.\n");
    }

    exit(0);
}

输出如下：

$ test
Here is child process.
Here is parent process.
child 4 is done.

虽然子进程最初与父进程拥有相同的内存内容，但父进程和子进程是在不同的内存和不同的寄存器中执行的：改变其中一个进程中的变量不会影响另一个进程。

exec系统调用使用新的内存映像来替换进程的内存，新内存映像从文件系统的文件中进行读取。当exec成功时，并不会返回到调用进程，而是在ELF头声明的入口点开始执行。exec需要两个参数，第一个是可执行文件的文件名，第二个是字符串参数数组。

int exec(char* file, char* argv[])

编写一个xv6用户应用，验证exec的功能：

#include "kernel/types.h"
#include "user/user.h"
#include "kernel/fcntl.h"

int main(void)
{
    char *argv[3];

    argv[0] = "echo"; 
    argv[1] = "hello world!"; 
    argv[2] = 0;

    exec(argv[0], argv);

    printf("exec() error\n");

    exit(0);
}

输出如下：

hello world!

1.2 I/O and File descriptors

文件描述符是一个小整数，代表了一个可由进程读取或写入的内核管理对象。

进程获得文件描述符的途径有：

  • 打开一个文件、目录、设备()open)

  • 复制一个文件描述符(dup、fork)

  • 创建管道(pipe)

xv6内核会为每一个进程维护一个以文件描述符为索引的表。一般来说，索引从0开始，0表示 标准输入，1表示 标准输出，0表示 标准错误。

关于read和write系统调用，read从文件描述符fd索引的文件中最多读取size个字节到缓存buf中，并返回读取的字节数，当返回值为0时，表示读到了文件尾部（EOF），当返回值为-1时，表示读取失败。

write从缓存buf读取size个字节，写入到文件描述符fd索引的文件中，并返回写入的字节数，当返回值为-1时，表示写入失败。

每个文件描述符都有一个偏移量，当读取或写入文件时，都会修改这个偏移量。

int read(int fd, char* buf, int size);

int write(int fd, char* buf, int size);

xv6的用户程序user/cat.c，实现了从标准输入读取字符串，写入到标准输出，核心代码如下

void
cat(int fd)
{
  int n;

  while((n = read(fd, buf, sizeof(buf))) > 0) {
    if (write(1, buf, n) != n) {
      fprintf(2, "cat: write error\n");
      exit(1);
    }
  }
  if(n < 0){
    fprintf(2, "cat: read error\n");
    exit(1);
  }
}

close系统调用会释放一个文件描述符，成功时返回0，失败时返回-1。open、pipe、dup等系统调用总是会重新分配当前进程的最小的文件描述符

int close(int fd);

open系统调用会打开一个文件，返回一个文件描述符：

int open(const char * file, int mode);

mode可以使用kernel/fcntl.h中定义的宏：

#define O_RDONLY  0x000
#define O_WRONLY  0x001
#define O_RDWR    0x002
#define O_CREATE  0x200
#define O_TRUNC   0x400

使用close、open系统调用和cat用户程序可以很容易地实现类似功能：cat < input.txt：

#include "kernel/types.h"
#include "user/user.h"
#include "kernel/fcntl.h"

int main(void)
{
    char *argv[2];

    argv[0] = "cat"; 
    argv[1] = 0;

    close(0);

    open("input.txt", O_RDONLY);

    exec(argv[0], argv);

    printf("exec() error\n");

    exit(0);
}

最终效果如下：

$ echo hello world > input.txt
$ cat < input.txt
hello world
$ test
hello world

虽然fork复制了文件描述符表，但每个底层文件的偏移量都是父子共享的。

#include "kernel/types.h"
#include "user/user.h"
#include "kernel/fcntl.h"

int main(void)
{
    if (fork() == 0)
    {
        write(1, "hello ", 6);
    }
    else
    {   
        wait(0);
        write(1, "world\n", 6);
    }

    exit(0);
}

最终效果：

$ test
hello world

综上所述，复制文件描述符(dup、fork)所产生的文件描述符会与被复制的文件描述符共享同一偏移量，其他情况(open、pipe)产生的文件描述符不会共享偏移量。

1.3 Pipes

pipe系统调用会创建一个管道，返回两个文件描述符p[0](读端)和p[1](写端)：

int pipe(int p[]);

在xv6上编写一个用户应用：

#include "kernel/types.h"
#include "user/user.h"
#include "kernel/fcntl.h"

#define WRITE 1
#define READ 0

int main(void)
{
    int p[2];
    pipe(p);
    char* argv[2];

    if (fork() == 0)
    {
        close(p[READ]);

        write(p[WRITE], "Hello World!\nSZU\n", 17);
        
        close(p[WRITE]);
    }
    else 
    {
        close(0);
        dup(p[READ]);
        close(p[WRITE]);

        argv[0] = "wc";
        argv[1] = 0;

        wait(0);

        exec(argv[0], argv);

        close(p[READ]);
    }

    exit(0);
}

运行效果如下：

$ test
2 3 17

1.4 File System

chdir系统调用会改变当前目录，mkdir系统调用会创建一个目录，mknod系统调用会创建一个引用设备的特殊文件，第二、第三个参数分别是主要设备号和次要设备号，两者可以唯一地标识一个设备：

int chdir(char* dir);
int mkdir(char* dir);
int mknod(char* file, int, int);

底层文件被称为inode，一个inode可以有多个名称，每个名称即为一个链接。每个链接由目录中的一个项组成；该项包含一个文件名和对 inode 的引用。

inode的信息如下(在kernel/stat.h中)：

#define T_DIR     1   // Directory
#define T_FILE    2   // File
#define T_DEVICE  3   // Device

struct stat {
  int dev;     // File system's disk device
  uint ino;    // Inode number
  short type;  // Type of file
  short nlink; // Number of links to file
  uint64 size; // Size of file in bytes
};

link系统调用会创建一个引用同一个文件file1（inode）的文件名file2：

int link(char* file1, char* file2);

unlink 系统调用会从文件系统中删除一个文件名。只有当文件的链接数为零且没有文件描述符引用它时，文件的inode和存放其内容的磁盘空间才会被释放。

int unlink(char* file);
