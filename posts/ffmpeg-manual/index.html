FFmpeg使用手册

FFmpeg 是一个开放原始码的自由软体，可以执行音讯和视讯多种格式的录影、转档、串流功能，包含了libavcodec——这是一个用于多个专案中音讯和视讯的解码器函式库，以及libavformat——一个音讯与视讯格式转换函式库。

From 维基百科，自由的百科全书

FFmpeg官网：https://www.ffmpeg.org/

FFmpeg官网文档：https://www.ffmpeg.org/documentation.html

简易教学视频：FFmpeg教程

下载与安装

下载地址：

  • https://www.ffmpeg.org/download.html#build-windows，点击Windows builds by BtbN

格式转换

使用如下命令可以把.flv文件转换为.mp4文件：

ffmpeg.exe -i P1.flv P1.mp4

通过-c:v(video encoder)参数，可以指定一个视频编码器：

ffmpeg.exe -i P1.flv -c:v libx264 P1.mp4

使用nVidia显卡的话，可以使用显卡加速：

ffmpeg.exe -i P1.flv -c:v h264_nvenc P1.mp4

在Windows上，可以使用time命令测试程序运行的时间（P2.flv大小为9.33 MB (9.792.007 字节)）：

$ time ffmpeg.exe -i P2.flv -loglevel error -c:v libx264 P2.mp4 -y
real    0m52.019s
user    0m0.000s
sys     0m0.031s

对于上述命令的理解：

ffmpeg.exe -i P1.flv -loglevel error -c:v h264_nvenc P1.mp4 -y 是一个使用FFmpeg工具的命令行指令。它有以下作用：

  1. ffmpeg.exe: 这是要执行的FFmpeg程序的文件名。
  2. -i P1.flv: 这表示输入文件的路径和名称。在这个例子中，输入文件是P1.flv。
  3. -loglevel error: 这个选项设置日志级别为错误级别，意味着只输出错误信息，而不显示其他详细信息。
  4. -c:v h264_nvenc: 这是视频编码器选项。h264_nvenc 是指使用 NVIDIA 的 NVENC 硬件加速器来进行 H.264 视频编码。这可以提高编码速度和效率。
  5. P1.mp4: 这是输出文件的路径和名称。在这个例子中，输出文件是P1.mp4。
  6. -y: 这个选项用于自动覆盖输出文件，如果已存在同名文件，则不会提示确认。

综上所述，该命令的作用是将输入文件 P1.flv 使用 H.264 编码器（利用 NVIDIA 的硬件加速）转换为输出文件 P1.mp4，并在转换过程中只显示错误级别的日志信息，并且如果输出文件已经存在，则自动覆盖。

通过-preset xxx可以指定编码速度，xxx为以下预设值：

  • ultrafast：最快的编码速度，但输出质量较低。
  • superfast：非常快的编码速度，输出质量稍高于 ultrafast。
  • veryfast：更快的编码速度，输出质量稍高于 superfast。
  • faster：更快的编码速度，输出质量稍高于 veryfast。
  • fast：较快的编码速度，输出质量稍高于 faster。
  • medium：中等编码速度和输出质量的平衡（默认值）。
  • slow：较慢的编码速度，输出质量更高。
  • slower：更慢的编码速度，输出质量更高。
  • veryslow：非常慢的编码速度，输出质量最高。

进行简单的测试：

$ time ffmpeg.exe -i P2.flv -loglevel error -c:v libx264 -preset ultrafast P2_bak.mp4 -y

real    0m37.708s
user    0m0.000s
sys     0m0.030s

$ time ffmpeg.exe -i P2.flv -loglevel error -c:v libx264 -preset veryslow P2.mp4 -y

real    1m39.915s
user    0m0.031s
sys     0m0.000s

$ ls -lh | grep ".mp4"
-rw-r--r-- 1 Lucas 197121 8.7M  7月 13 01:38 P2.mp4
-rw-r--r-- 1 Lucas 197121  18M  7月 13 01:40 P2_bak.mp4

不难发现，越快的编码速度，带来的是文件存储空间的增大。

通过-crf xxx (constant rate factor)可以控制图像质量，xxx取值为0(无损) - 51(最差质量) (常用19 - 28)：

$ time ffmpeg.exe -i P2.flv -loglevel error -c:v libx264 -crf 0 P2_bak.mp4 -y

real    1m4.081s
user    0m0.000s
sys     0m0.000s

$ time ffmpeg.exe -i P2.flv -loglevel error -c:v libx264 -crf 51 P2.mp4 -y

real    0m54.166s
user    0m0.000s
sys     0m0.015s

$ ls -lh | grep ".mp4"
-rw-r--r-- 1 Lucas 197121 5.0M  7月 13 01:49 P2.mp4
-rw-r--r-- 1 Lucas 197121  30M  7月 13 01:47 P2_bak.mp4

不难发现，越高的图像质量，带来的是文件存储空间的增大。

合并视频

使用ffmpeg命令可以把多个视频文件 按顺序 合并为一个视频文件:

首先，需要编写一个文件列表(myList.txt)：

file 'P1.flv'
file 'P2.flv'
file 'P3.flv'

$ time ffmpeg.exe -f concat -i myList.txt -loglevel error -c copy output.flv -y

real    0m0.405s
user    0m0.000s
sys     0m0.015s

对上述命令的理解：

这个命令使用FFmpeg工具来执行一系列操作。下面是每个选项的解释：

  • -f concat：指定输入文件的格式为concat，表示将合并多个输入文件。
  • -i myList.txt：指定一个文本文件（myList.txt）作为输入列表，该文件包含了要合并的视频或音频文件的路径和顺序。
  • -loglevel error：设置日志级别为错误级别，以减少输出的冗余信息。
  • -c copy：表示对输入文件进行复制操作，而不进行任何编码或转码操作。这可以有效地将源文件直接复制到输出文件中，避免重新编码所带来的质量损失和时间消耗。
  • output.flv：指定输出文件的名称为output.flv，这是合并后的视频或音频的最终结果。
  • -y：用于自动覆盖同名的输出文件，如果已经存在。
