ch02 Operating system organization 操作系统必须满足以下的需求： 多路复用（资源由进程共享） 隔离性（Isolation，进程之间不会互相影响） 交互性（进程之间可以互相通信） 2.1 Abstract physical resources 为了实现强隔离，操作系统应该禁止应用程序直接访问硬件资源，而提供对于硬件资源的抽象，如文件系统等（open、read、write、close）。 process：可以视为CPU的抽象 exec：可以视为内存的抽象 file：可以视为磁盘的抽象 2.2 User mode, supervisor mode, and system calls 强隔离性需要的硬件支持： user/kernel mode virtual memory RISC-V有三种模式： 机器模式 监督者模式 用户模式 ​ 机器模式主要用于配置计算机，启动CPU。在监督者模式（supervisor mode）下，CPU被允许运行特权指令（内核空间）。如果用户模式下的应用程序试图执行一条特权指令，CPU不会执行该指令，而是切换到监督者模式。应用程序只能执行用户模式的指令，如计算（用户空间）。 ​ 一个应用程序如果要调用内核函数（如xv6中的读系统调用），必须过渡到内核。 ​ CPU提供了一个特殊的指令ecall，可以让CPU从用户模式切换为监督者模式，并在内核指定的入口进入内核。 ​ CPU提供了一个特殊的指令sret，当系统调用完成后，内核切换回用户栈，并通过调用sret指令返回用户空间，降低硬件特权级别，恢复执行系统调用前的用户指令。 2.3 Kernel organization 宏内核 ：整个操作系统都在内核中，所有的系统调用都在监督者模式下运行。 微内核 ：在用户模式下运行操作系统的大部分代码。 从概念上讲，xv6是宏内核。 2.4 Code: xv6 organization xv6内核源码在kernel/子目录下。模块间的接口在kernel/defs.h中定义。 2.5 Process overview Xv6使用页表（由硬件实现）给每个进程提供自己的地址空间。 2.6 Code: starting xv6 and the first process
